# Spectralization 世界观与玩法设计

## 0. 这个模组到底在做什么

Spectralization 的工业幻想是：玩家不是把一串机器按配方排开，而是在基地里
建立一张光谱网络。光沿着端口、表面和场传播；机器的某个面因为涂层而变成
吸收面、反射面、输出镜或传感面；材料不是一张固定配方表，而是有谱线、有
损耗、有世界差异；终局的物品也不只是放在箱子里，而是挂在虚空信道上。

这就是“现代光学工业”在 Minecraft 里的版本。

## 1. Ver 1.0 的核心画面

一个毕业基地应该长这样：

- 屋顶有相控激光阵列。
- 中庭或地下有全息物质矩阵。
- 工业区关键机器都有工程化表面。
- 光纤、自由空间光路和微缩化光学模块共同组成网络。
- 发光方块被组织成光腔和泵浦结构。
- 光热塔把能量分给熔炉、反应釜和发电腔。
- 玩家坐在控制室里，用全息平板打开远处机器。
- 产线堵塞时，光点维护群沿着拓扑路径流向故障处。
- 虚空孔径阵列打开后，全息矩阵接入虚空存储总线。

这里的终局不是“箱子容量更大”，而是物品、机器、远程接口、光谱索引和虚空
信道被编成同一个工业系统。

## 2. 三条工业线

### 光热工业

前期让玩家明白一件事：光可以是工业能源。

基础玩法：

- 发光方块进入辐光收集器。
- 光热泵浦器把热光束打向机器。
- 机器表面的镀膜决定吸收哪种频率、反射哪种频率。
- 光热熔炉、热板、反应釜、发电腔通过光束运行。

这个阶段的味道是“我不再把加热器贴在机器旁边，我把热从远处打过去”。

### 光子信息

中期让玩家明白第二件事：光可以是控制网络。

基础玩法：

- 光纤和收发器连接工业区。
- 机器绑定器把具体机器接入全息平板。
- 平板权限分为 View、Touch、Slot、Matter。
- 光子指挥台显示拓扑、机器状态、订单流和维护任务。
- 光点维护群是 token 和客户端光点流，不是实体无人机海。

它不替代仓库网络。它做的是远程机器交互。

光纤在设定上不是 HUD 里才出现的透明光线，而是一条有包层、有接口、有可见
垂坠的导光线缆。玩家任何时候都应该能看见已经铺设的光纤；HUD 只负责显示
自由空间光束、诊断路径、信道读数和隐藏变量。

光纤内部接口是自由空间光路和光纤总线之间的边界。外部世界看到的是接口块和
端口方向；内部系统保存的是一条抽象导光连接。relay 只是线缆的几何锚点和寻
路节点，不是会参与求解的光学元件。这样光纤可以像工业线缆一样悬空、转向和
跨越距离，同时不破坏主光学编译器的线性模型。

导光匹配在玩法上表现为“能不能被接口接住”。光斑太大、发散太强或路线太长
时，未被导入的部分不会变成另一条可用光路，而是被粗略等效为接口附近的倏逝
场和包层损耗。玩家可以通过准直、聚焦、更好的接口、更短更直的路线或并行光纤
补偿这类损耗。

### 光谱物质

中后期让玩家明白第三件事：频率可以决定物质如何被处理和储存。

基础玩法：

- 可调谐激光扫描、切割、烧蚀、剥离涂层。
- 材料的谱线和阈值决定激光效果。
- 全息晶体把物品写成模板、数量、能量抵押和光谱索引。
- 全息物质矩阵按频率槽和信道稳定度扩容。
- 虚空存储总线把矩阵扩展到虚空信道。

这个阶段的储存不是“数字仓库”，而是光谱工程。

## 3. 红宝石激光器

红宝石激光器是第一个证明“规则能涌现机器”的系统。

当前设定：

- 萤石是弱非相干种子光。
- 被红石充能的萤石提供泵浦速率。
- 红宝石吸收合适的种子光，把它转换为相干红宝石线。
- 银块提供高反射背镜。
- 涂银玻璃提供输出耦合。
- 腔体强弱、红宝石数量、输出镜和泵浦位置共同决定输出。

它不是一个写死的多方块结构。玩家摆出来的是一个光学网络；编译器识别到反馈
腔、增益介质、输出口，然后让它工作。

## 4. 镀膜和表面

镀膜不是一个单独机器名，而是一类表面玩法。

前期表现：

- 用刷子和涂料给世界方块表面上涂层。
- 用砂纸删除涂层。
- 涂银玻璃这种有向方块直接把“某一面是银”做进模型和光学参数。

中期表现：

- 可镀膜物品在机器里被处理成带表面参数的物品。
- 镜子、透镜、滤光片、plate 有自己的涂层槽和品质。

后期表现：

- 可编程涂层。
- 超材料表面。
- 万能光学框架。

一个重要约定：涂层改变的是表面数据，不是把方块本身改写成另一种方块。破坏
普通方块通常不保留世界涂层；框架类方块会保留内部结构和 plate。

## 5. 场系统

沙子这类方块不是普通材料表的一行，而是 Field Source 的第一种例子。

场源的玩法意义：

- 它让某个空间区域的传播性质改变。
- 它可以让不可见光路在 HUD 或自然散射中显形。
- 它可以给某些机器提供环境条件。
- 它可以制造“光学地形”，让基地布局本身成为工程问题。

沙子现在做的是附近光路高损耗和可视化。未来同一套系统可以扩展到：

- 烟雾：强散射可见光。
- 热场：改变光热效率或损伤阈值。
- 虚空扰动场：影响虚空信道稳定度。
- 磁/电偏置场：影响偏振、调制或非线性晶体。
- 末影场：让 HUD 读取隐藏世界变量。

场源不是每 tick 扫一大片区域。它应该被编译成“哪些网络依赖这个场”的数据层
修正。

## 6. 随机材料系统

材料表不应该永远是一本公开手册。Spectralization 的长期乐趣之一，是同一种
材料在不同世界里有轻微谱线差异。

玩法结构：

- 数据包定义材料的基准谱线和响应范围。
- 世界 seed 给部分材料加入小偏移。
- 玩家用扫描器或光谱仪测量本世界材料。
- 测得的数据进入玩家的材料库。
- 镀膜、超透镜和高阶激光器会越来越依赖这些测量值。

这样，“测量”会成为工业过程的一部分，而不是百科查表。

## 7. 虚空存储总线

全息储存不是终点。终点是把全息矩阵接入虚空。

层级：

- 全息晶体：存模板、数量、能量抵押、光谱索引。
- 全息物质矩阵：管理本地晶体和信道。
- 虚空信道：把本地矩阵扩展到远处。
- 虚空存储总线：让多个矩阵、接口和孔径共享一个虚空寻址空间。
- 虚空孔径阵列：建立、稳定、放大虚空信道。

虚空总线的重点不是“无限容量”四个字，而是：

- 信道分配。
- 稳定度。
- 能量抵押。
- 远程物质接口。
- 模板重构。
- 虚空资源抽取。
- 与全息平板和指挥台联动。

后期玩家管理的不是箱子，而是一张跨距离、跨机器、跨信道的物质网络。

## 8. 微缩化结构

微缩化结构是把“编译器也是玩法”这件事真正放进游戏。

设想：

- 玩家在工作站里搭一个二维光路。
- 工作站把这个光路编译成 macro node。
- 激光扭曲空间，把这套光路压进一个方块或一个机器核心。
- 这个核心对外暴露多个逻辑端口，端口数量可以超过六个面。
- 外部网络只看到它的等效 scattering map。

这会让玩家自己设计机器。也会解释为什么后期设备必须使用超透镜和微缩化结构：
不是因为世界里搭不出来，而是因为体积、稳定度和端口密度不允许。

## 9. 空间频率与 HUD

现实中空气里的光束通常不可见。游戏里要给玩家反馈，所以分两种可视化：

- 自然可视化：灰尘、烟雾、电离、荧光、spot。
- HUD 可视化：末影材料、调试头盔、后期光学眼镜。

皮革帽子现在只是 debug 设备。正式路线应该逐步过渡到末影 HUD，因为末影系
材料可以合理地“看到”光束、信道和世界隐藏变量。

空间频率系统提供：

- 光束半径。
- 发散角。
- 聚焦距离。
- spot size。
- 散杂度。
- 光纤耦合参数。
- 光热 irradiance。

“烂光束”会进入广义散杂/非相干通道。它仍然有能量，也会产生 spot 和热，但
不会像干净的相干光那样参与精密反馈。

## 10. 超透镜

超透镜是后期的设计型玩法。

玩家选择目标超透镜，再调整材料预算。预算给出一组可实现包络；目标参数落入
包络，则设计成功。机器记住目标，合成时消耗材料，模板不消耗。

这借用了逆向设计的味道，但不要求玩家真的解方程。玩家做的是材料预算、目标
参数和工业成本之间的权衡。

## 11. 横向实用设备与跨维光学

这一组设想不是当前版本的实现承诺，而是给后续横向设备保留方向。它的目标是
让光学参数不仅服务机器配方，也能改变世界、空间和物流路线。

核心链路：

```text
频谱工程
-> 选择性破坏
-> 维度开孔
-> 光桥物流
-> 跨维加工
```

选择性破坏不应该只是“激光镐”。方块可以有带色散的破坏阈值：不同频率、相干
度、光斑和脉冲形态对应不同的有效破坏能量。选择合适频率可以更省能量地破坏
某类材料；选择错误频率则只会产生热、散射或无效损耗。反过来，防护材料也可以
针对特定频率吸收、反射或散射。基岩也可以被放进同一套系统：它不是普通意义上
“硬度无限”，而是只有极窄、极苛刻的共振窗口。合适频率的高质量脉冲可以成为
官方破基岩路线，而不是绕过规则的漏洞。

稳定传送门可以被看作跨维度的光学边。玩家在某个位置用满足条件的光维持一个
孔径，实体、物品和光都可以与它互动。对光学求解器来说，它应当像一个带频率
条件、相干条件、损耗和维度映射的传输元件：

```text
portal input face
-> dimension mapping
-> portal output face
```

传送门不应该是一旦点火就永久存在的装饰物。它应由光维持：功率、频率、相干性
或稳定度不足时逐渐衰减。它可以加载目标区块，但必须有数量、能量和权限边界。
这样玩家可以把主世界的光打进下界或末地，也可以把另一个维度的处理结果带回
本地工业线。

光桥是这条线的实体交互层。它不是普通方块桥，而是由相干光或特定投影器维持的
可碰撞/可引导结构。光桥可以引导实体和物品运动，也可以把物品送过稳定传送门。
某些物品穿过传送门时发生相变；某些配方则要求材料连续经过不同维度中的机器，
例如先在主世界预处理，再进入下界掺杂，最后去末地稳定化。

这条路线的设计原则：

- 频率选择、光斑质量和相干性必须有可见后果。
- 破坏、传送门和光桥都应从现有光学规则涌现，而不是写成孤立魔法方块。
- 光能跨维度传播，但必须显式消耗稳定度、加载预算或孔径资源。
- 跨维加工应该强化维度差异，而不是把维度机器做成普通配方台的换皮。
- 所有自动区块加载、基岩破坏和实体传送都必须走明确的权限、保护和调试日志。

## 12. 项目边界

这些边界不是退让，而是为了让模组有清楚的形状：

- 光学系统服务玩法，不服务完整电磁场仿真。
- 远程机器交互和虚空总线有自己的价值，不抢传统仓库网络的位置。
- 大宗物流仍然可以交给其他模组。
- 无人机群是光点流和任务 token，不是实体海。
- 脚本化逻辑会做成卡片、模块和条件槽，而不是开放式电脑。
