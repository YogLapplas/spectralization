# Spectralization

Spectralization 是一个 NeoForge 1.21.1 科技模组实验。它想做的不是“又一套
机器链”，而是一种少见的工业体验：玩家用光谱、端口、镀膜、反馈腔、场源、
全息储存和虚空信道来设计自己的基地。

现在的原型已经走过了最早的试验阶段。它有了第一代光网络编译器，有了能在
世界里自然形成的红宝石激光器，也有了涂层、光束、spot 可视化、光纤网络、
热力熔炼、第一版全息存储矩阵和微缩机器框架。接下来要把这些基础收束成真正的中期玩法：
微缩化光路工作站、超材料/光刻路线，以及它们通向远程机器交互和虚空
存储总线的路线。

## 当前能玩的东西

- 创造光源（含频谱柱状编辑器）、镜子、动态镜、分光镜、透镜架、CMOS、贯通传感器、光束分析仪、光谱仪。
- 红宝石块、银块、涂银玻璃组成的初级固体激光腔。
- 金/银涂料桶、砂纸、高级刷子、创造刷子和世界表面涂层。
- 真理套装占位物品；真理头盔和皮革帽暂时都能承担 debug 可视化：能看到涂层、光束拓扑和 surface spot。光束与光斑共用频率到 RGB 的轻量 LUT。
- Jade 显示部分传感器读数。
- 光束伤害、致盲、荧光管、沙尘场等早期互动。
- 光纤系统第一版：接口、relay、自动寻路连接、光输入输出、并行光纤容量涌现、剪断光纤和 Jade 连接数显示。
- 光热发电机、环境光收集器、热力熔炼炉和一批熔融流体。
- 全息存储矩阵第一版：核心/水晶/screen 多方块识别、容量由水晶与暴露面决定、存储终端、核心信息 UI 和内置合成台。
- 微缩机器第一版：anchor/core/io_port 框架识别、工作区域可视化、压缩输出物品，以及压缩后方块对世界光路的面到面映射和主动光源输出。
- 环境光收集器使用平滑谱族 LUT：原版光源仍只通过环境采样进入散杂光，不作为全局内禀光源；创造光源默认可见白光连续谱。
- 第一代 port graph 编译器：能识别网络、调度求解、处理反馈、缓存读数，并按活跃频率 lane 并行求解多频率光束。

## 现在的技术方向

光学系统按三层运行：

- 几何层：世界里有哪些光学节点、方块、表面、场源。
- 拓扑层：这些节点的端口如何连接。
- 数据层：材料、涂层、增益、频率、功率、读数和可视化导出量。

光源 tick 不负责重算光路。它只申请缓存或应用缓存。真正的重建和求解进入
预算队列；高优先级变化可以中断低优先级导出计算。这个结构是后续所有玩法的
底座，包括微缩化光路、全息存储矩阵、虚空存储总线和随机材料系统。

当前开发主线已经从“先把全息存储跑起来”转向“微缩化光路”。微缩机器不是普通
机器外壳，而是把一段局部光路、端口图和缓存求解结果压缩成一个 macro node。
压缩后的方块仍然是光学元件和可能的光源，因此可以继续参与更外层的压缩：
玩家先在更易操作的工作区里学习光学结构，再把它们装进可接入世界光网的机器。

开发规范也按这个方向收束：新光源、新传感器、新可视化和新材料行为都要先接入
拓扑编译、事件驱动和缓存导出。旧 tracer 只保留为 debug oracle 和迁移期对照，
不能作为新功能的主运行路径。

## Debug Log

光学编译器日志可以通过 config 或游戏内命令开启：

```text
logs/spectralization/optical_compiler_<timestamp>_UTC.log
```

常见诊断行：

```text
network_cache ...
solver_plan ...
gain_schedule ...
receiver_outputs ...
system_receiver_outputs ...
spot_export ...
beam_overlay ...
```

这些日志不是错误报告，而是我们的实验仪器。它们用来检查网络是否被正确识别、
求解器是否按计划运行、反馈腔是否被稳定调度、传感器读数是否可靠。
