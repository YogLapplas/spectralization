# 凝结的时空 / Condensed Spacetime

凝结的时空是 Spectralization 黑洞时代之后的终局物品。它不是单纯的强力装备，而是一种把光学、虚空、全息、FE、物品、机器拓扑和局部世界规则折叠到同一个端口对象里的时空材料。

它有两种主要使用形态：

- 装备态：直接作用于玩家、玩家持有物品、远程访问和施工行为。
- 放置态：作为终局中枢方块，强化附近 Spectralization 系统，并给复杂网络提供稳定性和带宽。

## 装备态设想

- 相位存在：玩家 hitbox 近似不存在，获得绝对隐身语义。实现时只能保证 Spectralization 和常规实体交互层面，不能保证所有模组都完全忽略玩家。
- 瞬间破坏方块：必须走标准破坏事件、权限、保护和掉落逻辑。
- 触及距离 +10。
- 快捷键停止时间：不能安全冻结整个服务器世界。更适合实现为冻结 Spectralization 系统、局部实体/投射物或短时局部 tick 门控。
- 区块非实体数据记录与复制：记录一个区块的方块与 BlockEntity 数据，在另一个区块消耗充能复制。必须有黑名单、容量限制、维度校验和能量成本。
- 带宽式无限能量：给身上物品提供无限能量，但每 tick 吞吐受带宽限制。
- 虚空存储零消耗访问：远程访问虚空存储不再消耗能量。
- 创造飞行。
- 抗性提升 20。
- 边界重映射：让两个绑定的方块面在 IO、FE、光学、物品交互上视为相邻。对 Spectralization 光学可完全支持；对 FE/物品通过 capability 代理；对任意模组私有逻辑不保证。
- 相位施工：先放置无碰撞、半透明蓝图方块，之后统一提交材料实体化。
- 物品世界线缓存：只记录单个物品栈的 NBT、配置、耐久和组件状态，不做世界区域回滚。

## 放置态设想

- 降低附近机器访问虚空存储的能耗。
- 带宽式无线无限充能：给附近消耗能量的机器供能，总吞吐受带宽限制。
- 拓扑折叠场：降低范围内机器、光纤、虚空访问、FE 传输等逻辑距离损耗。
- 局部 tick 排程：只作用于 Spectralization 机器，允许局部指定更新顺序，减少竞态。
- 局部边界条件编辑：只影响 Spectralization 系统，例如光路损耗、散热、虚空访问损耗、全息带宽、机器稳定性。
- 光学系统全局缓存加速：作为终局级光学/全息网络中枢，提高附近复杂系统的稳定性和刷新效率。
- 全息晶体网络中枢：强化全息存储、无人机临场、虚空存储与黑洞封装相关系统。
- 黑洞反应堆稳定核心：用于稳定黑洞时代的高阶发电与奇异材料处理。

## 需要提前预留的接口

### SpectralAccessContext

所有 Spectralization 内部访问行为都应能携带一个上下文对象。它不直接执行功能，而是告诉系统当前访问发生在什么边界条件下。

典型字段：

- costMultiplier
- bandwidthLimit
- distanceMultiplier
- stabilityBonus
- heatMultiplier
- voidAccessMultiplier
- holographicBandwidthMultiplier
- opticalLossMultiplier

这个上下文应被 FE、虚空存储、全息存储、光纤、机器访问、光学网络和微缩机器复用。

### BandwidthLedger

带宽式无限能量和虚空访问都不应是真正的无限吞吐。它们应该由同一个账本系统记录每 tick 可用带宽。

典型用途：

- 玩家装备态给物品充能。
- 放置态给附近机器无线供能。
- 虚空存储远程访问。
- 全息晶体网络吞吐。
- 光纤/远程 GUI 控制带宽。

### BoundaryLinkRegistry

记录两个 SurfaceKey 或方块面的绑定关系。任何系统在寻找邻接面时，都可以先查询是否存在边界重映射。

第一阶段只需要支持：

- 光学端口重映射。
- FE capability 代理。
- 物品 capability 代理。

未来扩展：

- 流体。
- 热量。
- 光纤信道。
- 虚空信道。

### SpectralFieldProvider

凝结的时空放置态应该作为一种强场源。它和沙子扰动场、未来热场、虚空场、拓扑折叠场使用同一套场接口。

典型导出量：

- opticalLossScale
- topologyDistanceScale
- voidAccessCostScale
- holographicBandwidthScale
- machineStabilityBonus
- solverBudgetBonus

### ChunkSnapshotService

为区块复制预留，不急于实现。

必须避免：

- 复制实体。
- 复制玩家。
- 复制跨维度非法状态。
- 复制会造成无限资源漏洞的运行中机器状态。

### PhaseBlueprintService

为相位施工预留。

核心流程：

1. 放置蓝图方块。
2. 记录目标方块状态和所需材料。
3. 允许玩家调整、撤销或移动蓝图。
4. 提交时统一消耗材料并实体化。

### ItemTimelineCache

只作用于单个 ItemStack。

允许记录：

- NBT / data components。
- 耐久。
- 模式配置。
- 内部能量。

不允许记录：

- 世界区域状态。
- 实体状态。
- 任意时间回滚。

## 实现原则

1. 凝结的时空可以改写 Spectralization 的规则，但不能假设能安全改写整个 Minecraft。
2. 对其他模组的联动优先通过事件、能力接口和代理层完成。
3. “无限”必须以带宽、冷却、稳定性、热量或虚空成本的形式受限。
4. 放置态优先做系统中枢，装备态优先做玩家体验。
5. 它应该复用现有三层架构：几何层、拓扑层、数据层，而不是绕过编译器。
